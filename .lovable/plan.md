# AI & Answer-Engine Optimization for PhotoBrief

Goal: make PhotoBrief discoverable and quotable by ChatGPT, Claude, Perplexity, Gemini, and code agents (Cursor) — not just Google. No backend work needed; everything is static + a small reusable hook layer. Existing `SEOHead` already handles per-route title/description/canonical/OG/Twitter/JSON-LD via direct DOM mutation, so we extend that pattern rather than introducing react-helmet.

---

## 1. Discovery files in `/public`

New static files (served at the root of all three custom domains):

- **`/public/llms.txt`** — short markdown:
  - One-paragraph canonical definition of PhotoBrief ("PhotoBrief is an AI-guided visual intake tool that converts a single SMS/email link into a complete, quality-checked photo brief…").
  - Pricing summary table (Free / Starter / Pro / Business — pulled from `planLimits.ts`).
  - Comparison table vs. generic alternatives (email photo dumps, Typeform + uploads, Jotform, custom intake forms).
  - 3 example queries with quotable answers ("How does PhotoBrief handle blurry photos?", "Do recipients need an app?", "What does the API return?").
  - Links to every other discovery file below + `/for-ai-agents`, `/pricing`, `/help`, `/sitemap.xml`.
- **`/public/llms-full.txt`** — extended markdown: full feature list (mirrors `FeatureKey` in `planLimits.ts`), the entire FAQ corpus from `src/features/help/content/faq.tsx` + `Pricing.tsx`, and a condensed API reference for `POST /api-create-request`.
- **`/public/.well-known/ai-plugin.json`** — ChatGPT plugin manifest pointing at the OpenAPI spec, with name, description, contact email, legal URL.
- **`/public/.well-known/agent.json`** — agent capabilities manifest (name, description, capabilities array: `create_request`, `lookup_pricing`, `read_faq`; auth = `bearer_token`; endpoints).
- **`/public/openapi.json`** — OpenAPI 3.1 spec covering the live edge function `POST https://app.photobrief.ai/functions/v1/api-create-request` (request body, 200/400/401 responses, bearer auth scheme). Generated by hand from `supabase/functions/api-create-request/index.ts`.
- **`/public/mcp.json`** — MCP server descriptor advertising a future MCP endpoint and listing the same tool surface as `agent.json`. Marked `"status": "planned"` so we don't lie about a live endpoint.
- **`/public/robots.txt`** — extend current allow-list to explicitly allow: `GPTBot`, `OAI-SearchBot`, `ChatGPT-User`, `ClaudeBot`, `Claude-Web`, `anthropic-ai`, `PerplexityBot`, `Perplexity-User`, `Google-Extended`, `Applebot-Extended`, `CCBot`, `Bytespider`, `meta-externalagent`. Each as its own `User-agent` block with `Allow: /`.
- **`/public/sitemap.xml`** — add `/for-ai-agents`, `/signup`, `/waitlist` entries; bump `lastmod` to today.

### Cloudflare Worker routing

## 2. Structured data (JSON-LD) — single source of truth

- **Sitewide Organization + WebSite** — added once in `index.html` as a static `<script type="application/ld+json">` block (so it appears in the prerendered HTML for every route without React work). Includes `name`, `url`, `logo`, `sameAs`, `potentialAction` (SearchAction).
- **Home (`/`)** — `SoftwareApplication` JSON-LD (already partially present via `LANDING_JSONLD`). Extend with `aggregateRating` placeholder removed, `featureList`, `screenshot`, `softwareVersion`, `offers` array generated from `planLimits.ts` so pricing stays in sync.
- **FAQPage** — built dynamically in a new `useFaqJsonLd(items)` hook. The `Landing` and `BetaGuidePage` pages call it with the **same** `faqItems` array they already render visibly — no duplication. The hook strips React nodes to plain text via a small `extractText()` helper.
- **BreadcrumbList** — new reusable `useBreadcrumbJsonLd(crumbs)` hook. Called from every marketing page (`/`, `/pricing`, `/help`, `/for-ai-agents`, `/signup`, `/waitlist`).
- **HowTo** — added on `/for-ai-agents` and the home page using the existing 3-step `HowItWorksSteps` array as the source (export the `steps` array from that component so the JSON-LD reads from the same data).
- **Article** — applied to `/help` (`BetaGuidePage`) using its existing TOC/sections.

All JSON-LD goes through the existing `SEOHead` `jsonLd` prop (already supports an array), so there's one injection path.

## 3. Reusable `usePageMeta` hook

Today every page calls `<SEOHead title="…" description="…" canonicalPath="…" jsonLd={…} />`. We'll keep `SEOHead` (it works) and add a thin `usePageMeta()` hook on top:

```ts
usePageMeta({
  title,           // enforced ≤ 60 chars at runtime in dev (console.warn)
  description,     // enforced ≤ 160 chars
  canonicalPath,
  ogImage,         // defaults to /og-image.jpg
  ogType,          // 'website' | 'article' | 'product'
  twitterCard,     // defaults to 'summary_large_image'
  jsonLd,          // array, merged with sitewide blobs
  breadcrumbs,     // optional — auto-emits BreadcrumbList JSON-LD
});
```

Internally it just calls `SEOHead`. All marketing pages migrate to it. Adds the missing `og:image:width/height`, `og:site_name`, `twitter:site`, `twitter:creator` tags currently absent.

### `og-image.jpg`

Generated locally via a one-off script using the existing brand mark (`src/assets/brand/photobrief-horizontal.png`) on the brand gradient background, exported at 1200×630 JPG, written to `/public/og-image.jpg`. The existing `og-image.png` stays as a fallback.

## 5. `/for-ai-agents` landing page

New route in `App.tsx` under `MarketingLayout`: `/for-ai-agents` → `ForAiAgentsPage`.

Sections:

1. **Hero** — "PhotoBrief for AI agents" + one-paragraph canonical definition.
2. **Quotable facts grid** — 8 short declarative sentences, each in its own `<article>` with an `id` so agents can deep-link.
3. **Comparison table** — PhotoBrief vs. Typeform / Jotform / generic upload form, rendered as a real `<table>` (agents parse tables well).
4. **Code samples** — 3 tabs: cURL, JavaScript (`fetch`), Python (`requests`) hitting `POST /functions/v1/api-create-request`. Pulled from a single `apiExamples.ts` module so docs and JSON-LD `HowTo` stay in sync.
5. **MCP / agent setup** — short instructions + JSON snippets for `agent.json` and `mcp.json` consumers, with copy buttons.
6. **Discovery endpoints** — bulleted list linking to every file from §1.
7. **FAQ** — reuses `faqItems` (same source as `/help`), so JSON-LD and visible HTML can never drift.

Adds the page to `sitemap.xml`, `llms.txt`, internal nav (footer link "For AI agents"), and emits `Article` + `HowTo` + `FAQPage` + `BreadcrumbList` JSON-LD via `usePageMeta`.

## Technical notes

- No backend changes required. The existing `api-create-request` edge function is the only thing the OpenAPI/MCP descriptors need to describe.
- Prerender (`scripts/prerender.mjs`) will pick up the new hook output on `/`, `/pricing`, `/help`, `/for-ai-agents` automatically because `usePageMeta` runs in `useEffect` (same model as today's `SEOHead`).
- Single source of truth enforcement:
  - FAQ → `src/features/help/content/faq.tsx` (already exists; `Pricing.tsx`'s local `faqs` array is migrated into it, tagged `audience: "business"`).
  - Pricing/plan facts → `src/config/planLimits.ts`.
  - How-it-works steps → exported from `HowItWorksSteps.tsx`.
  - API examples → new `src/config/apiExamples.ts`.
- All new hooks live in `src/hooks/seo/` (`usePageMeta.ts`, `useFaqJsonLd.ts`, `useBreadcrumbJsonLd.ts`, `useHowToJsonLd.ts`).
- Dev-only `console.warn` if title > 60 chars or description > 160 chars to prevent regressions.
